    📖 Explicación del proyecto

    1. Cómo se ha construido la web

    La web está hecha en un único archivo HTML que incluye dentro el CSS (estilos) y el JavaScript (interacción). No se conecta a ninguna base de datos: es una simulación visual del proyecto Python original que sí usa SQLite.

    2. Estructura HTML utilizada

    • <header>: encabezado con el título y descripción.
    • <main> / <section>: cada bloque (formulario, filtros, lista, explicación) está dentro de una section.
    • <form>, <input>, <button>: formulario para añadir tareas con descripción y prioridad.
    • <ul> y <li>: lista donde se muestra cada tarea como un elemento.
    • <footer>: pie de página con la información final.

    3. Estilos CSS aplicados

    • Paleta de colores suave: definida con variables CSS (--primary, --success, etc.).
    • Tipografía: fuente del sistema, limpia y legible.
    • Distribución: flexbox para alinear formulario, filtros y tareas.
    • Bordes redondeados (border-radius) y sombras suaves (box-shadow) en tarjetas.
    • Espaciado coherente con padding y margin.
    • Diseño responsive: con @media (max-width: 600px) la interfaz se adapta a móvil.
    • Estados visuales: tareas completadas en verde y tachadas; prioridades con colores (verde→rojo según urgencia).

    4. Qué hace cada parte (equivalencia con Python + SQLite)

    • Formulario "Añadir tarea": en el proyecto real ejecutaría un INSERT INTO tareas en SQLite con descripción y prioridad.
    • Filtro "Ver pendientes": equivale a SELECT * FROM tareas WHERE completada = 0.
    • Filtro "Ver completadas": equivale a SELECT * FROM tareas WHERE completada = 1.
    • Botón "Completar": ejecutaría UPDATE tareas SET completada = 1 WHERE id = ?.
    • Botón "Eliminar": ejecutaría DELETE FROM tareas WHERE id = ?.
    • Lista de tareas: representa los datos que en Python vendrían de una consulta SELECT a la tabla tareas (id, descripcion, prioridad, completada).

    5. JavaScript usado

    Solo se usa JavaScript para simular la interacción en el navegador: añadir, completar, eliminar y filtrar tareas. Los datos se guardan en memoria (un array), no en una base de datos real.
